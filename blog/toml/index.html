toml文档格式的数据定义

md文件配置规范

February 2, 2020 么了

1 分钟阅读

  • Twitter

  • Facebook

  • LinkedIn

  • Email

TOML　的全称是　Tom’s Obvious, Minimal Language，因为它的作者是 GitHub　联合创始人　Tom Preston-Werner 。

  • TOML是大小写敏感的
  • # 表示注释
  • 可以按照自己的意愿使用空格或者Tab缩进

先写一个 Hugo 博客/md文档的配置规范

+++
title = "标题名称"
description = "" //副标题
draft = false //草稿
author = "么了"
date = "2019-04-02"
categories = ["笔记",博客""] // 设置分类，支持数组格式
series = ["A", "B"]
slug = "url的尾部"
tags = []
keywords: ["A", "B"]
featured = "pic01.jpg" //外部索引图片地址用文件夹 /年份/月份/pic.jpg
featuredalt = "Pic 1"
featuredpath = "date" // 也可使用绝对路径
linktitle = ""
publishdate      // 定时未来发布的时间
type = "post"
metaDataFormat = "toml"
languageCode = "en-us"
date = "2014-09-29T21:54:53-05:00"
weight         // 排序的权重，默认按时间降序

+++

Hugo 的 markdown 文件前置数据定义结构，支持三种格式的语法类型

例如：

yaml

---
title: "My First Post"
date: 2019-03-26T08:47:11+01:00
draft: true
---

toml

+++
title: "My First Post"
date: 2019-03-26T08:47:11+01:00
draft: true
+++

json

{
	"title": "{{ replace .Name "-" " " | title }}",
	"date": {{ .Date }},
	"draft": true
}
